## 2. Kế hoạch tháo gỡ (Unblocking Roadmap)

Để mở khóa hoàn toàn việc phê duyệt thiết kế Discovery Engine, thứ tự ưu tiên xử lý các rào cản được đề xuất như sau:

```mermaid
graph TD
    subgraph Phase A: Founder Decisions (Quyết định cốt lõi)
        BLK-OQ5[BLK-OQ5: Mismatch Logic] --> BLK-OQ12-13[BLK-OQ12-13: Weighting Models]
        BLK-DOM-01[BLK-DOM-01: Concurrency Control]
    end

    subgraph Phase B: Domain & Schema Alignment (Đồng bộ Domain & Database)
        BLK-DEP-01[BLK-DEP-01: Align ClaimedSkillArea in Domain] --> BLK-SCHEMA-01[BLK-SCHEMA-01: Reconcile DDL Schema]
    end

    subgraph Phase C: API & Prompts Alignment (Đồng bộ Hợp đồng & Prompt)
        BLK-SCHEMA-01 --> BLK-DEP-02[BLK-DEP-02: Prompt Input Contract]
        BLK-SCHEMA-01 --> BLK-API-01[BLK-API-01: Fill Missing Endpoints]
        BLK-API-01 --> BLK-DEP-03[BLK-DEP-03: Canonical Output Layers]
    end

    subgraph Phase D: Polish & Merge (Dọn dẹp và Tối ưu)
        BLK-DEP-03 --> BLK-MERGE-01[BLK-MERGE-01: Merge State Machine]
        BLK-MERGE-01 --> BLK-DRIFT-01[BLK-DRIFT-01: Clear Outdated Risks]
    end

    Phase A --> Phase B
```

*Chú thích: Việc giải quyết xong Phase A (Founder chốt thiết kế giải pháp cho OQ5, OQ12/13, và Concurrency) sẽ tự động tháo gỡ các blocker về mặt logic nghiệp vụ, tạo điều kiện thuận lợi cho việc cập nhật chéo đồng bộ kỹ thuật ở các Phase tiếp theo.*
