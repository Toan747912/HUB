# AI Documentation Index

Entry point for every document covering the AI subsystem (`Apps/ai-backend`): ADRs, architecture/integration reviews, audits, certifications, technical standards, and planning documents. If a new AI document is written and it is not linked from this file, it does not count as published.

## 1. AI Documentation Overview

The AI subsystem is a NestJS backend (`Apps/ai-backend`) built around two layers:

- **AI Brain** (certified, WP-AI-02) — five single-shot planners (Mission, Discovery, Knowledge, Evidence, Teaching), each a thin subclass of `BasePlannerService`, reachable through `AiRuntimeService` (`POST /ai/execute`). Shared infrastructure: `ContextAssemblyService`, `ResilientLlmGateway` + `RedisCircuitBreakerService`, `ExplainabilityRulesService`, structured logging/metrics/audit.
- **Agent Layer** (certified CONDITIONAL PASS, WP-AI-03/04) — a multi-step, stateful, tool-using layer (`agent-runtime`, `agent-tools`, `agent-memory`, `agent-lifecycle`, `agent-coordinator`, `agent-message-bus`, `agent-collaboration`, `agent-learning`, `agent-core`) built as an architectural seam on top of the AI Brain, without modifying the five planners or `BasePlannerService`. See [AgentLayerCertification.md](AgentLayerCertification.md).

All documents below describe reviews, audits, certifications, or decisions about these two layers — none of them are the source of truth for behavior; the code in `Apps/ai-backend/src` is.

## 10. Navigation Rules

Every future AI document added under `12_AI/` must:

1. **Belong to one category** — ADR, Review, Audit, Certification, Standard, or Plan. State the category in the document's opening lines.
2. **Have an owner** — named individual or team responsible for the document's accuracy going forward.
3. **Have a status** — e.g. Proposed / Accepted / PASS / CONDITIONAL PASS / FAIL / Planning Only / Superseded.
4. **Reference its work package** — the `WP-AI-NN` (or sub-phase, e.g. `WP-AI-02 Phase 2`) it belongs to.
5. **Reference related ADRs** — any ADR that constrains or motivates the document (e.g. an audit of the Agent Layer should reference `ADR-061`).
6. **Be linked from this README** — add it to the relevant index/table in Sections 3–9 in the same change that adds the document.
